# 먹BTI 설계 문서

## 개요

먹BTI는 MBTI에서 영감을 받은 식습관 유형 검사 서비스다. 사람들이 음식에 대해 가지는 취향과 행동 패턴을 4가지 이진 축으로 분류하고, 결과를 통해 식사 궁합이 맞는 유형을 찾을 수 있도록 돕는다.

핵심 목적: 먹BTI가 비슷한 사람끼리 식사하면 즐겁고, 다른 유형끼리는 어떤 충돌이 생기는지 알 수 있게 한다.

---

## 1. 유형 체계

### 4가지 축

| 축 | A형 코드 | A형 설명 | B형 코드 | B형 설명 |
|---|---|---|---|---|
| 식사 가치관 | **G** (Gourmet, 미식형) | 맛과 퀄리티가 최우선 | **P** (Practical, 실용형) | 빠르고 편한 것이 최우선 |
| 식당 결정 방식 | **J** (계획형) | 미리 검색·예약·동선 준비 | **S** (Spontaneous, 즉흥형) | 걷다가 눈에 띄면 입장 |
| 음식 도전 성향 | **E** (Explorer, 탐험형) | 새 메뉴·새 식당 계속 시도 | **R** (Regular, 단골형) | 검증된 곳·검증된 메뉴만 |
| 맛 취향 | **H** (Hot, 자극형) | 맵고 짜고 강한 맛 선호 | **M** (Mild, 순한형) | 담백하고 깔끔한 맛 선호 |

### 16가지 유형 코드 및 별명

| 코드 | 별명 | 코드 | 별명 |
|---|---|---|---|
| GJEH | 예약 맛집 탐험가 | GJEM | 예약 맛집 순례자 |
| GJRH | 단골 맛집 고집러 | GJRM | 단골 맛집 미식가 |
| GSEH | 즉흥 맛집 헌터 | GSEM | 즉흥 미식 탐험가 |
| GSRH | 불금 단골집 왕 | GSRM | 편안한 미식 추구형 |
| PJEH | 효율 탐험 계획러 | PJEM | 건강 맛집 계획형 |
| PJRH | 점심 루틴 고수 | PJRM | 조용한 점심 계획형 |
| PSEH | 즉흥 도전 실용형 | PSEM | 가볍게 먹는 탐험형 |
| PSRH | 퇴근길 단골 마니아 | PSRM | 아무거나 잘 먹는 형 |

## 2. 퀴즈 설계

### 기본 원칙

- 총 20문항 (축당 5문항)
- 각 문항은 A/B 이진 선택
- **간접 질문 원칙**: 어떤 축을 측정하는지 티가 나지 않도록 상황 기반 질문으로 설계. 실제 심리검사처럼 응답자가 결과를 예측하기 어렵게 함
- 일부 문항은 두 축을 동시에 측정할 수 있음

### 문항 예시

**G/P 축 (미식 vs 실용):**
> "점심시간이 30분밖에 없다. 10분 거리에 줄 서는 맛집이 있고, 2분 거리에 무난한 식당이 있다. 당신은?"
> A) 뛰어서라도 맛집 간다 → G
> B) 가까운 데서 빠르게 먹는다 → P

**J/S 축 (계획 vs 즉흥):**
> "친구가 '오늘 저녁 뭐 먹을까?' 물어봤을 때 당신이 먼저 하는 행동은?"
> A) 폰을 꺼내 지도앱을 켠다 → J
> B) 일단 나가서 걸으면서 본다 → S

**E/R 축 (탐험 vs 단골):**
> "좋아하는 식당에서 오늘 처음 보는 신메뉴를 발견했다. 당신은?"
> A) 무조건 시켜본다 → E
> B) 원래 먹던 걸 시킨다 → R

**H/M 축 (자극 vs 순한):**
> "편의점에서 라면을 고른다면?"
> A) 불닭볶음면 계열 → H
> B) 진라면 순한맛 계열 → M

### 점수 계산

각 축에서 A 응답 수 / 전체 문항 수 = A형 비율. 예: 계획형 문항 5개 중 4개 선택 → 계획형 80% / 즉흥형 20%.
최종 유형은 각 축에서 50% 초과한 쪽으로 결정.

## 3. 결과 페이지

### 구성

**① 유형 코드 + 별명**
```
🔥 GJEH — 예약 맛집 탐험가
```

**② 축별 비율 바 시각화**
```
미식 ████████░░ 80%  |  실용 20%
계획 ██████░░░░ 60%  |  즉흥 40%
탐험 █████████░ 90%  |  단골 10%
자극 ███████░░░ 70%  |  순한 30%
```

**③ 유형 설명 (풀어쓴 텍스트)**
각 16가지 유형마다 2~3문단 분량의 서사적 설명. 예:
> "당신은 먹기 위해 사는 진정한 미식 탐험가입니다. 새로운 식당 오픈 소식에 가장 먼저 반응하고, 여행지를 정할 때 맛집 리스트가 숙소보다 먼저 완성되어 있죠. 강렬한 맛을 사랑하고, 즉흥적인 상황에서도 맛집을 귀신같이 찾아냅니다..."

**④ 궁합 유형**
- 잘 맞는 유형 2개: 공유 축이 많은 유형
- 충돌 가능성이 있는 유형 1개: 핵심 축(G/P)이 반대인 유형

### 궁합 알고리즘

G/P 축(식사 가치관)을 가장 중요한 축으로 설정. 이 축이 같으면 기본적으로 궁합이 좋음. 나머지 3축 중 2개 이상 일치하면 "잘 맞는 유형", G/P 축이 반대이면 "충돌 가능성" 표시.

## 4. 기술 스택

| 항목 | 선택 | 이유 |
|---|---|---|
| 프레임워크 | React + TypeScript | 퀴즈 상태 관리 적합 |
| 스타일링 | Tailwind CSS | 빠른 UI 구성 |
| 배포 | Vercel | 무료, 간단 |
| 공유 | 카카오 공유 SDK + URL 파라미터 | 결과 URL에 유형 코드 포함 (`/result?type=GJEH`) |

백엔드 및 DB 없음. 순수 프론트엔드로 동작.

## 5. 페이지 구성

| 페이지 | 경로 | 내용 |
|---|---|---|
| 홈 | `/` | 먹BTI 소개, 시작 버튼 |
| 퀴즈 | `/quiz` | 문항 하나씩 순서대로, 진행 바 표시 |
| 결과 | `/result?type=GJEH` | 유형 코드, 비율 바, 설명, 궁합, 공유 버튼 |

---

## 6. 성공 기준

- 20문항 퀴즈 완주 후 올바른 4글자 유형 코드가 생성됨
- 결과 URL을 공유했을 때 동일한 결과 페이지가 열림
- 16가지 유형 각각에 설명 텍스트가 존재함
- 카카오톡 공유 버튼이 동작함
